Working with script files in R Studio

To work with R, it is possible to type commands directly into the console and see the results. However, this is not a good way to work with R, because:

  1. You would need to repeat this process to run the analysis again.
  2. Your analysis is not transparent - no one else can see what you did.

Instead, it is better to write scripts that contain the commands that you want to run and save them, so they can be reused and shared with others.

Create a folder for your data and scripts

Before you start, create a folder on your computer to store your data and scripts. This will make it easier to find your files later. You can call this folder anything you like, but it is a good idea to give it a name that is related to your project. For example, for this session, you might create a folder called Practical1.

NoteTask

Create a new script file

To create a new script file, click on the File -> New File -> R Script. This will open a new window, which is where you will write your script. It is important to save your script as you work. To do this, click on the File -> Save As and give your script a name. It is a good idea to save your script in the same folder as your data.

Set the working directory

The working directory is the folder that R will look in for files. To set the working directory, click on the Session -> Set Working Directory -> Choose Directory and select the folder you created for your data.

If you already have a script open and saved, you can also set the working directory by clicking on the Session -> Set Working Directory -> To Source File Location. This will set the working directory to the folder where your open script is saved.

NoteTask

Set the working directory to the folder you created for your data and scripts.

ImportantWhy do we need to set the working directory?

When you run a command in R (such as importing a dataset), it will look for files in whichever folder is set as the working directory. If you do not set the working directory, R will look in the default location, which is not usually where your files are. Therefore, it is important to set the working directory to the folder you are using for your data and scripts. This will make it easier to work with R, importing and exporting data.

Adding comments to a script

It is possible to add information to a script that will not be run as a command. This is called a comment. Comments are useful for adding information about what you are doing in your script. To add a comment, you need to start the line with a hash symbol (#). Anything after the hash symbol will be ignored by R. For example:

# This is a comment

Writing commands / code in a script

In an R script, any text that is not a comment is assumed to be R code. R Code are instructions that tell R to do something.

For example:


1# Add two numbers together

223 + 52
1
This is comment that will be ignored by R. It is used to describe what the code does.
2
This code is a command that tells R to add 23 and 52. However, it will not do anything until you tell R to run it.

Saving objects in R

When you run a command in R, you can save the results to an object. For example, when you run the command to add two numbers together, you can save this object by assigning it to a name. For example:


# Store a set of numbers in an object

1numbers <- c(23, 52)

# Add two numbers together and store the result in an object

2result <- 23 + 52
1
This code tells R to create a vector of numbers (23 and 52) and save it to an object called numbers.
2
This code tells R to add 23 and 52 and save the result to an object called result. The result of the code on the right side of the <- is assigned to the object on the left side of the <-.

You can save many things as objects including the results of analyses, data, and graphs. You will see stored objects in the environment pane in R Studio.

TipThe <- assignment operator

Note how the code above uses <- to assign the result to an object. This is called the assignment operator. It is created using the keys < and - with no space in between them.

It is important to use <- to assign objects in R. If you use =, it will work, but it might cause problems later on.

Using functions in R

A function is a command that performs a specific task by providing it with certain information (called “arguments”).

For example, the function mean() calculates the mean of a set of numbers.

To use a function, you need to provide it with some information. For example, to calculate the mean of a set of numbers, you need to tell the function which numbers to use. For example:

# Calculate the mean of my previous result

1mean(result)
1
In the code above, the function mean() is used to calculate the mean of the object result.

The object result (which was created earlier) is the argument for the function mean().

Importing data into R

ImportantDownload the data for this section

For this section you will need to download the following dataset: practical1_data.csv

Right click and select Save link as… to download the file. Save it in the same folder as your script.

To import data into R, you can use the a command that relates to the type of data you are importing. For example, to import the data in the file practical1_data.csv, you would use the following command:

1data <- read.csv("practical1_data.csv")
1
This code tells R to import the data in the file practical1_data.csv and save it to an object called data. The object data could be named anything you like, but it is a good idea to give it a name that is related to the data you are importing.

You can also use the menu to import data. To do this, click on the Import Dataset button and select the type of data you want to import. For example, to import the data in the file practical1_data.csv, you would click on From Text (readr) and then select the file practical1_data.csv.

Viewing data in R

To view the data in an object, you can use the View() function. For example, to view the data in the object data, you would use the following command:

1view(data)
1
When you run this code, you should see the dataset in a new window of R Studio.
NoteTask

View the data in the object data.

Tip

Notice that the data is displayed in rows and columns. Each row represents a participant and each column represents a variable. This type of data structure is called a data frame.

To refer to a specific column in a data frame, you can use the dollar sign ($) followed by the name of the column. For example, to refer to the column age in the data frame data, you would use the following command:

1data$age
1
When you run this code, you should see the column age from the data frame data printed in the console.

You can use data frame columns in functions. For example, to calculate the mean of the column age in the data frame data, you would use the following command:

1mean(data$age)
1
When you run this code, you should see the mean of the column age from the data frame data printed in the console.
TipDon’t overwrite objects accidentally

When you save objects in R, you assign them a name and use the <- operator. However, if you use the same name more than once, this will overwrite the existing object. For example, this code contains an error. Can you see what it is?:


1mean_age <- mean(data$age)
2mean_age <- mean(data$id)
1
The object mean_age is assigned the mean of the column age in the data frame data.
2
However, the next line of code assigns the mean of the column id in the data frame data to the object mean_age. This overwrites the existing object mean_age.
